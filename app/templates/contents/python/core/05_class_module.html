class 내부에서 함수를 생성시에 반드시 self(== this)를 인수로 받아야함
생성자 함수
def __init__(self, first, second)
  self.first = first

형태로 지정함

사용은 a = FourCal(4, 2) 처럼함

class FourCal:
  def __init__(self, first, second):
    self.first = first
    self.second = second
  def setdata(self, first, second):
    self.first = first
    self.second = second
  def add(self):
    result = self.first + self.second
    return result
  def mul(self):
    result = self.first * self.second
    return result
  def sub(self):
    result = self.first - self.second
    return result
  def div(self):
    result = self.first / self.second
    return result
a = FourCal() # 생성자가 있으면 애러남
b = FourCal() # 생성자가 있으면 애러남
a.setdata(4, 2)
a.add()
b.setdata(3, 8)
b.add()

상속

새로만든 class의 (parent class) 형태로 지정해서 사용할 수 있음
class MoreFourCal(FourCal):
  def pow(self):
    result = self.first ** self.second
    return result
a = MoreFourCal(4, 2)
a.pow()

overriding

부모 클래스에 있는 메서드를 동일한 이름으로 다시 만들어서 사용할 수 있음
class SafeFourCal(FourCal):
  def div(self):
    if self.second == 0:
      return 0
    else:
      return self.first / self.second
  def sub(self):
    if self.first < self.second:
      self.first, self.second = self.second, self.first
    return self.first - self.second
a = SafeFourCal(4, 0)
a.sub()

//mod1.py
def add(a,b):
    return a+b
def sub(a,b):
    return a-b
add(3,4)
sub(4,2)

위와 같이하면 import 시 자동으로 실행됨
shell에서 import mod1을 하면 add(3,4)와 sub(4,2)가 실행됨
이를 방지하기 위해

//mod1.py
def add(a,b):
    return a+b
def sub(a,b):
    return a-b
if __name__ == "__main__":
  add(3,4)
  sub(4,2)

위와 같이 하면 shell에서 python mod1.py를 하면 if name == “main” 이 참이되어 실행되고,
import mod1을 하면 실행되지 않음
